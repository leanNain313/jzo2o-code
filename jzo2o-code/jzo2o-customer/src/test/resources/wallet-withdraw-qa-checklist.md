# 钱包提现回归与联调清单

## 后端关键场景
- [ ] 发起提现：余额充足，钱包余额扣减成功，生成提现申请（状态=0）与提现账单（type=2）
- [ ] 发起提现：余额不足，接口返回失败，钱包与账单无变化
- [ ] 审核：`0 -> 1`（通过进入打款中），记录审核人和审核时间
- [ ] 审核：`0 -> 3`（拒绝），失败原因必填，余额返还并生成收入账单（type=0）
- [ ] 审核：`1 -> 2`（打款成功），状态变更成功，不返还余额
- [ ] 审核：`1 -> 3`（打款失败），失败原因必填，余额返还并生成收入账单（type=0）
- [ ] 非法流转校验：`0 -> 2`、`2/3 -> 其他` 均应失败
- [ ] 并发提现校验：同钱包并发提交不会超扣（`balance >= amount` 条件更新生效）
- [ ] 支付成功入账：订单支付后钱包入账一次，生成收入账单（按 serviceOrderId 幂等）

## 服务人员端联调
- [ ] “我的”页面出现“我的钱包”入口
- [ ] 钱包页可展示余额
- [ ] 可发起提现申请（缺银行卡信息时有提示）
- [ ] 账单可分页展示，含类型/金额/时间/描述
- [ ] 点击账单可跳转订单详情（serviceOrderId 存在时）

## 运营端联调
- [ ] 菜单新增“金融管理”
- [ ] “用户提现申请审核管理”可分页查询、按状态筛选
- [ ] 可执行通过、拒绝、打款成功、打款失败四种操作，状态变更符合规则
- [ ] “用户钱包明细”可分页查询钱包列表
- [ ] 点击“查看账单”可弹窗查看对应钱包账单明细
